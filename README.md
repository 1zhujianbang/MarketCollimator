# MarketCollimator

## 项目概述

MarketCollimator 是一个专注于全球交易品价格数据可视化呈现的系统。项目旨在提供清晰、准确、实时的多维度市场数据展示，帮助用户直观理解全球交易品价格走势与分布，**不包含任何价格分析、预测或交易建议功能**。

## 项目定位

- **纯粹可视化工具**：专注于数据呈现，无分析逻辑
- **多市场覆盖**：支持股票、外汇、商品、加密货币等交易品种
- **实时/历史数据**：提供时间维度的灵活数据展示
- **开发者友好**：模块化设计，便于扩展和维护

## 核心功能

### 数据可视化模块
- **时间序列图表**：K线图、折线图、面积图展示价格走势
- **多品种对比**：同一图表中展示多个相关交易品价格
- **市场热度图**：按板块、地域等维度呈现价格变化热力图
- **分布直方图**：特定时间点的价格分布情况
- **交互式探索**：缩放、平移、数据点查看等交互功能

### 数据源管理
- **多数据源适配**：支持API、数据库、文件等多种数据源
- **数据清洗管道**：标准化不同来源的数据格式
- **缓存机制**：优化频繁访问数据的性能

### 视图定制系统
- **仪表板配置**：用户可自定义可视化组件布局
- **预设模板**：提供常用的视图模板（如大盘概览、品种聚焦等）
- **显示参数调整**：时间范围、周期、颜色方案等可配置

## 技术架构

### 前端层
- **可视化框架**：基于ECharts/D3.js的图表组件
- **UI框架**：React/Vue + TypeScript
- **状态管理**：Redux/Pinia
- **响应式设计**：支持桌面端、平板、移动端

### 后端层
- **API网关**：RESTful/GraphQL接口
- **数据服务**：数据获取、转换、缓存服务
- **任务调度**：定时数据更新任务

### 数据层
- **实时数据流**：WebSocket支持实时价格推送
- **历史存储**：时序数据库（如InfluxDB）或传统关系型数据库
- **缓存层**：Redis/Memcached

## 开发指导原则

### 可视化原则
1. **准确性优先**：数据呈现必须准确反映源数据
2. **清晰性**：避免视觉混乱，保持图表简洁易懂
3. **一致性**：相同类型的图表保持统一的视觉风格
4. **性能优化**：大数据量下的流畅交互体验

### 代码规范
1. **模块化设计**：每个可视化组件独立可复用
2. **配置驱动**：图表样式、数据映射通过配置实现
3. **类型安全**：全面使用TypeScript接口定义数据格式
4. **文档完整**：所有组件和API需有清晰文档

### 数据中立性
- **严禁**在代码中添加任何分析逻辑（如趋势判断、支撑阻力位计算）
- **严禁**添加任何形式的交易建议或信号提示
- **保持**数据呈现的客观性，不强调特定数据点

## 扩展开发指南

### 添加新的可视化类型
1. 在`visualizations/`目录下创建新组件文件夹
2. 实现基础可视化接口
3. 添加配置选项定义
4. 编写示例和文档

### 接入新的数据源
1. 在`data/connectors/`中创建新连接器
2. 实现标准数据获取接口
3. 添加数据清洗和转换逻辑
4. 更新数据源配置选项

### 创建新的视图模板
1. 在`views/templates/`中定义新模板
2. 配置可视化组件布局
3. 设置默认参数
4. 添加到模板选择器

## 开发路线图

### 第一阶段：基础框架
- 核心可视化引擎开发
- 基础图表组件（K线、折线图）

### 第二阶段：功能扩展
- 多数据源支持

- 性能优化（大数据量渲染）
- 自定义主题系统
- 插件化架构
- 协作功能（视图分享）

## 注意事项

1. **保持项目纯粹性**：坚决抵制添加分析功能的请求
2. **数据源可靠性**：明确标注数据来源和更新频率
3. **性能监控**：大规模数据渲染时的性能考量
4. **浏览器兼容性**：根据目标用户确定支持范围
